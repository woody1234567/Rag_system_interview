# Fubon MA Interview - Task Summary

## 我對任務的理解

這份任務要我建立一個針對「富邦金控 113 年報」的 RAG 問答系統，並且不只要能回答問題，還要可被驗證、可量化評估、且能處理幻覺（hallucination）風險。

根據提供的 PDF / 圖片說明與題庫 Excel，我理解面試評分重點如下：

1. **基本分（必做）**
   - 建置可運作的 RAG 系統
   - 說明架構設計與方法邏輯
   - 使用題庫測試，並以標準答案比對
   - 計算整體正確率（Accuracy）
   - 回答需提供來源依據（頁碼或段落）

2. **幻覺檢測與排除（重點）**
   - 定義什麼是幻覺（例如：無依據編造、超出文件範圍推論）
   - 建立判斷機制（是否有證據、證據是否充分）
   - 對「年報未揭露」題目必須**正確拒答**，不能硬猜

3. **加分方向（差異化）**
   - 更好的特徵/檢索/建模策略
   - 對結果有幫助的前處理或 reranking 方法
   - 具有商業分析價值的 insight

## 任務產出方向（我後續會這樣做）

1. **資料準備**
   - 解析年報 PDF，保留頁碼、段落、表格資訊
   - 清理文字（斷行、全半形、數字格式）

2. **RAG 系統建置**
   - Chunking：以語意 + 頁碼切塊
   - Embedding + 向量檢索
   - Top-k 召回 +（可選）reranker
   - 生成時強制引用來源

3. **答案驗證機制**
   - 對照 Excel 標準答案
   - 指標：Accuracy（可再加 Exact Match / F1）
   - 產出每題結果：
     - 問題
     - 模型答案
     - 標準答案
     - 是否正確
     - 來源頁碼

4. **幻覺管控機制**
   - 若檢索證據不足：回覆「資料不足，無法推論」
   - 增加回答置信門檻（evidence threshold）
   - 對拒答題檢查是否成功拒答

5. **加分嘗試**
   - 數值題做結構化抽取與運算
   - 跨頁題做多跳檢索（multi-hop）
   - 類型化分析錯誤（查找錯 / 推理錯 / 幻覺錯）
